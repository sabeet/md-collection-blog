Started by creating this project using **React** and **Vite** using just plain Javascript.

Was pretty straight forward. The command for that was:

```powershell
 $ npm create vite@latest <INSERT-NAME-OF-YOUR-BLOG-HERE>
```

cd into the project directory

```powershell
$ cd  <INSERT-NAME-OF-YOUR-BLOG-HERE>
```

Afterwards, I added this project to github. Make sure you have a repo initialized in Github for this.

```powershell
$ git init
$ git add .
$ git commit -m "Initial commit"
$ git branch -m main
$ git remote add origin https://github.com/sabeet/<INSERT-NAME-OF-GITHUB-REPO-HERE>.git
$ git push -u origin main
```

I didn't install these next things all at once but they most certainly would help me now if I had to redo this website.

```powershell
npm install react-router-dom framer-motion react-markdown @tailwindcss/typography
```

Next, Tailwind CSS v4 was added via its official Vite plugin.

```powershell
npm install tailwindcss @tailwindcss/vite
```

In `vite.config.js`, add the Tailwind plugin alongside the default React one:

```js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [
    react(),
    tailwindcss(),
  ],
})
```

Then in `index.css`, replace everything with:

```css
@import "tailwindcss";
@plugin "@tailwindcss/typography";
```

---

Posts are written as markdown files and stored in a separate public GitHub repo. Each file is numbered (`01.md`, `02.md`, etc.) and includes frontmatter at the top:

```
---
title: My Post Title
date: 2025-06-01
---
Your markdown content here
```


The blog fetches the list of posts using the GitHub Contents API, then fetches each file individually from `raw.githubusercontent.com`. Both happen in parallel using `Promise.all` so the page loads as fast as possible.

Since frontmatter parsing libraries like `gray-matter` wasn't working for some reason, a small custom utility called `ParseFrontmatter` was written. It uses a regex to split the `---` block from the markdown body and parses each `key: value` line into an object.

---
The blog logo was converted from text to SVG path data using this tool:

[https://danmarshall.github.io/google-font-to-svg-path/](https://danmarshall.github.io/google-font-to-svg-path/)

The path data was stored in a constants file and animated using Framer Motion's `pathLength` feature allowing animated tracing letter strokes on page load. A CSS glitch effect using `drop-shadow` in red and cyan fires on hover.

Page transitions between routes are handled by wrapping each page in a `PageTransition` component that uses Framer Motion's `motion.div` with a fade animation. `AnimatePresence` in `main.jsx` listens for route changes and triggers the exit before the new page mounts.
